# ./a11y — политика reduced-motion

> Роль: справка — публичный API экспорт-субпутя `./a11y`: фабрика `createMotionConfig` — глобальная политика `prefers-reduced-motion` с режимами `'system' | 'always' | 'never'`, реактивной подпиской и синтезированным `matchMedia`-швом для остальных субпутей движка.

## Назначение

Субпуть `./a11y` — политика reduced-motion уровня приложения: аналог класса поведения `MotionConfig` у Motion и `Globals.skipAnimation` у react-spring, но без глобального синглтона. Конфиг — обычное значение: приложение само решает, один он на всё дерево или свой на каждый скоуп.

Ключ интеграции — синтезированный `matchMedia`-шов. Каждый исполнитель движка (`drive`, `./gestures`, `./decay`, `./flip`, `./presence`, `./lit`, …) принимает инжектируемый `matchMedia` и читает через него `(prefers-reduced-motion: reduce)`. Вместо сырого `window.matchMedia` передаётся `cfg.matchMedia` — и всё семейство запросов `prefers-reduced-motion` начинает отражать ПОЛИТИКУ (режим + системное предпочтение), а не голую систему. Политика включается одной строкой на вызов, без изменения ядра; `cfg.set('always')` выключает движение во всём приложении.

Инварианты пакета (закреплены в JSDoc и тестах):

- **A1. Zero-DOM / SSR-safe.** Модуль не читает `window`; системный `matchMedia` — инжектируемый шов. Без него режим `'system'` даёт `false` (движение включено), `'always'`/`'never'` работают полностью.
- **A2. Прозрачное проксирование.** Запросы вне семейства `prefers-reduced-motion` без изменений уходят в системный шов.
- **A3. Zero runtime deps, ноль модульного глобального состояния.** Конфиг — значение; никаких скрытых синглтонов.

Характеристики размера — не в этой странице: снимок чисел даёт вывод `pnpm size`, свод — в [docs/benchmark.md](../benchmark.md).

## Импорт

```ts
import {
  createMotionConfig,
  type MotionConfig,
  type MotionConfigOptions,
  type ReducedMotionMode,
} from '@labpics/motion/a11y';
```

`createMotionConfig` — единственный runtime-экспорт субпутя.

## API

### createMotionConfig

```ts
function createMotionConfig(options?: MotionConfigOptions): MotionConfig;
```

Создаёт политику reduced-motion. Не бросает — LM-кодов у субпутя нет; временных параметров (миллисекунд) тоже нет.

Опции (`MotionConfigOptions`):

- `reducedMotion?: ReducedMotionMode` — начальный режим. По умолчанию `'system'`. Любое невалидное значение молча нормализуется в `'system'`.
- `matchMedia?: (query: string) => MediaQueryList` — системный шов; в браузере `window.matchMedia.bind(window)`, `undefined` = SSR/Node.

Если системный шов передан, конфиг при создании подписывается на `change` у `(prefers-reduced-motion: reduce)`: через `addEventListener`, с fallback на legacy `addListener`/`removeListener`. Если шов бросил при подписке, исключение глотается — политика остаётся опрашиваемой, но без реактивности от системы.

Возвращает `MotionConfig` со следующими членами.

#### prefersReduced()

```ts
prefersReduced(): boolean;
```

Эффективное предпочтение с учётом режима: `'always'` → `true`; `'never'` → `false`; `'system'` → `matches` системного `(prefers-reduced-motion: reduce)` (`false` при отсутствии шва или его исключении).

#### set(mode)

```ts
set(mode: ReducedMotionMode): void;
```

Сменить режим на лету. Подписчики уведомляются только если сменилось ЭФФЕКТИВНОЕ булево значение (например, `set('always')` при уже редьюсящей системе — тишина). Невалидный `mode` — no-op: ни режим, ни эффект не меняются.

#### onChange(cb)

```ts
onChange(cb: (reduced: boolean) => void): () => void;
```

Подписка на смену эффективного значения — edge-triggered: колбэк зовётся только при фактическом изменении, будь то `set()` или системное событие в режиме `'system'`. Возвращает функцию отписки.

#### matchMedia(query)

```ts
matchMedia(query: string): MediaQueryList;
```

Синтезированный шов — передавайте в любой API движка вместо `window.matchMedia`:

- Всё семейство `prefers-reduced-motion` отражает политику согласованно: `reduce` и голая форма `(prefers-reduced-motion)` → `matches === prefersReduced()`, `no-preference` → инверсия. Противоречие «reduce=true И no-preference=true» невозможно.
- Возвращаемый MQL для reduce-семейства — статический СНИМОК на момент вызова: `addEventListener`/`addListener` на нём — no-op, `dispatchEvent` → `false`. Для реактивности используйте `onChange()`.
- Прочие запросы прозрачно проксируются в системный шов (A2); без шва или при его исключении — статический MQL с `matches: false`.

#### destroy()

```ts
destroy(): void;
```

Отпускает системного слушателя и очищает всех подписчиков. Обязателен для короткоживущих конфигов — иначе системный `MediaQueryList` удерживает слушателя навсегда (утечка). После `destroy()` конфиг остаётся опрашиваемым (`prefersReduced()`/`matchMedia()` работают), но уведомлений больше нет.

#### mode

```ts
readonly mode: ReducedMotionMode;
```

Текущий режим (readonly-геттер); отражает последний успешный `set()`.

### Type-only экспорты

- `ReducedMotionMode` — `'system' | 'always' | 'never'`.
- `MotionConfigOptions` — опции `createMotionConfig` (см. выше).
- `MotionConfig` — форма возвращаемого конфига: `prefersReduced`, `set`, `onChange`, `matchMedia`, `destroy`, `mode`.

## Контракты

- **SSR-safe (A1).** Ни `window`, ни DOM, ни часов; шов инжектируется. `createMotionConfig()` без опций в Node не бросает; `'system'` без шва = движение включено.
- **Прозрачность (A2).** Не-reduce запросы неотличимы от прямого вызова системного `matchMedia`.
- **Нет глобального состояния (A3).** Ноль runtime-зависимостей; два конфига полностью независимы.
- **Edge-triggered уведомления.** `onChange` зовётся только на смене эффективного булева; порядок — синхронно из `set()`/системного события, по снимку списка подписчиков.
- **Робастность.** Исключения системного шва глотаются (`prefersReduced()` → `false`, проксирование → статический `matches: false`); невалидный режим в конструкторе → `'system'`, в `set()` → no-op. Сам субпуть никогда не бросает.
- **Reduced-motion — исполнение на стороне движка.** `./a11y` ничего не анимирует: реакция на политику — контракт исполнителей (`drive` при reduce не входит в солвер и синхронно резолвится одним `onStep(to)`; жесты/decay снапают аналогично).

## Примеры

Политика приложения одной строкой на вызов: `drive` с `cfg.matchMedia` вместо сырого `window.matchMedia`:

```ts
import { drive } from '@labpics/motion';
import { createMotionConfig } from '@labpics/motion/a11y';

const cfg = createMotionConfig({
  reducedMotion: 'system',
  matchMedia: typeof window !== 'undefined' ? window.matchMedia.bind(window) : undefined,
});

const box = document.querySelector('.box') as HTMLElement;

async function open(): Promise<void> {
  await drive({
    from: 0,
    to: 240,
    spring: { mass: 1, stiffness: 200, damping: 20 },
    onStep: (x) => {
      box.style.transform = `translateX(${x}px)`;
    },
    matchMedia: cfg.matchMedia, // политика, не голая система
  });
}

void open();

// Тумблер настроек приложения: выключить движение везде.
cfg.set('always'); // следующий open() снапнет мгновенно: onStep(240) один раз
```

Реактивный индикатор + жест, уважающий политику; демонтаж через `destroy()`:

```ts
import { createMotionConfig } from '@labpics/motion/a11y';
import { createDrag } from '@labpics/motion/gestures';

const cfg = createMotionConfig({ matchMedia: window.matchMedia.bind(window) });

const knob = document.querySelector('.knob') as HTMLElement;
const drag = createDrag({
  axis: 'x',
  bounds: { x: { min: 0, max: 320 } },
  matchMedia: cfg.matchMedia, // инерция отпускания уважает политику
  onStep: (x) => {
    knob.style.transform = `translateX(${x}px)`;
  },
});
knob.addEventListener('pointerdown', (e) =>
  drag.pointerDown({ x: e.clientX, y: e.clientY, t: e.timeStamp }),
);
knob.addEventListener('pointermove', (e) =>
  drag.pointerMove({ x: e.clientX, y: e.clientY, t: e.timeStamp }),
);
knob.addEventListener('pointerup', (e) =>
  drag.pointerUp({ x: e.clientX, y: e.clientY, t: e.timeStamp }),
);

// UI-индикация эффективного значения — реактивно через onChange.
const badge = document.querySelector('.rm-badge') as HTMLElement;
const render = (reduced: boolean): void => {
  badge.textContent = reduced ? 'reduced' : 'full motion';
};
render(cfg.prefersReduced());
const unsub = cfg.onChange(render);

// Короткоживущий скоуп обязан отпустить системного слушателя.
window.addEventListener('beforeunload', () => {
  unsub();
  cfg.destroy();
});
```
