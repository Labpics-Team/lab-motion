# Закрытая форма пружины: полюсное пространство и точность

> Роль: справка — объяснение математики спрингового ядра: почему @labpics/motion решает пружину замкнутой формой в полюсных координатах, как устранена catastrophic cancellation через тождество Виета (#226) и как `springAsEasing` запечатывает хвост C¹ Hermite-коррекцией (#219).

## Назначение

Эта страница — объяснение («почему так»), а не справка по API: полный перечень сигнатур субпутя `./spring` — в [docs/reference/spring.md](../reference/spring.md). Здесь — математика, стоящая за публичным поведением: чистым солвером `spring()` (корневой субпуть) и конструкторами/easing-проекцией субпутя `./spring`.

Нормализованная задача ядра (демпфированный гармонический осциллятор, тянущийся к цели):

```text
m·x'' + c·x' + k·x = k,   x(0) = 0,   x'(0) = v0
```

`from = 0`, `to = 1` — масштабирование в реальный диапазон делает вызывающий. Публичный `spring()` — частный случай `v0 = 0`; произвольный `v0` использует smooth pickup у `MotionValue`. Время `t` — **секунды**.

Три свойства, которые страница объясняет:

1. **Замкнутая форма вместо интегратора** — произвольный доступ по `t`, детерминизм, аналитическая валидация.
2. **Полюсное пространство и точность (#226)** — почему коэффициенты решения считаются через `α`, `ω²`, `Δ` и резольвентное тождество Виета, а не через классическую запись `−ω₀(ζ − √(ζ²−1))`.
3. **Конечный easing-горизонт и Hermite-хвост (#219)** — как бесконечно затухающая кривая проецируется на `t ∈ [0, 1]` без endpoint-прыжка.

Характеристики размера/производительности — не в этой странице: снимок чисел даёт вывод `pnpm size` / `pnpm bench`, свод — в [docs/benchmark.md](../benchmark.md).

## Математика

### ζ и ω₀: безразмерные координаты режима

Из тройки `{mass: m, stiffness: k, damping: c}` выводятся две канонические координаты:

```text
ω₀ = √(k/m)          — собственная частота (рад/с)
ζ  = c / (2·m·ω₀)    — коэффициент демпфирования (безразмерный; = c/(2√(km)))
```

Режим определяет только `ζ`:

- `ζ < 1` (`c < 2√(km)`) — underdamped: колебания с перелётом;
- `ζ = 1` — критическое демпфирование: быстрейший подход без перелёта;
- `ζ > 1` — overdamped: «плоский» медленный подход;
- `ζ = 0` (`damping = 0`) — незатухающая: `x = 1 − cos(ω₀·t)`, не оседает никогда.

Отсюда точная **масштабная инвариантность**: тройки `(λm, λk, λc)` дают те же `ζ` и `ω₀` — ту же кривую. Ею пользуется `springAsEasing`: его горизонт зависит только от `ζ`, поэтому scale-equivalent параметры получают одну easing-кривую.

### Полюсное пространство: α, ω², Δ

Характеристическое уравнение задачи — `r² + 2α·r + ω² = 0` с полюсами `r = −α ± √(α² − ω²)`. Солвер (`internal/solver.ts`) считает в канонической pole-space форме (#226):

```text
α  = c/m/2      — скорость затухания (деление ДО половинения: 2·m
                  переполняется при конечных scale-equivalent m/k/c)
ω² = k/m
Δ  = ω² − α²    — дискриминант со знаком
```

Ветвление — по знаку `Δ`: `> 0` underdamped, `== 0` critical, `< 0` overdamped. Обе стороны near-critical **непрерывно сходятся** к критической форме, поэтому branch-политике не нужен magic-epsilon — граница честная, `Δ === 0`.

Решения по ветвям (формулы солвера; для публичного `spring()` подставьте `v0 = 0`; при `t ≤ 0` возвращается `x = 0`, `x′ = v0`):

**Underdamped (`Δ > 0`)** — комплексные полюса `−α ± i·ω_d`:

```text
ω_d = √Δ,   B = (v0 − α)/ω_d
x(t)  = 1 + e^(−α·t)·(B·sin(ω_d·t) − cos(ω_d·t))
x′(t) = e^(−α·t)·(−α·(B·sin − cos) + ω_d·(sin + B·cos))
```

**Critical (`Δ = 0`)** — двойной полюс `−α`:

```text
B = v0 − α
x(t)  = 1 + (B·t − 1)·e^(−α·t)
x′(t) = e^(−α·t)·(B − α·(B·t − 1))
```

**Overdamped (`Δ < 0`)** — два вещественных полюса; форма описана в разделе про #226 ниже, потому что её запись целиком продиктована точностью.

### Почему замкнутая форма, а не интегратор

Численный интегратор (Эйлер, RK4, полунеявные схемы физдвижков) шагает по времени: результат в момент `t` — функция всей траектории шагов до него. Замкнутая форма — чистая функция `(params, t) → {value, velocity}`. Это даёт четыре свойства, на которых стоит остальной пакет:

1. **Произвольный доступ по `t`.** Кадровому циклу не нужен «предыдущий кадр»: пропущенные кадры, tab-throttling, seek таймлайна — просто другое `t`. Горячий путь компилятора сэмплирует одну пружину сотнями значений `t` на сетке (`makeSpringValueSampler` — внутренняя фабрика с хойстом петле-инвариантов) — для интегратора это сотни полных прогонов.
2. **Детерминизм.** Идентичные `(params, t)` → идентичный выход, независимо от частоты кадров и порядка вызовов. Инвариант заявлен в шапке `spring.ts` и является контрактом, а не следствием удачного шага.
3. **Ноль накопленной ошибки.** У интегратора ошибка растёт с числом шагов и зависит от `dt`; у аналитического решения точность каждой точки определяется только floating-point-оценкой самой формулы — поэтому вся борьба за точность сосредоточена в форме записи (см. #226), а не в размере шага.
4. **Аналитическая валидация.** Верхняя граница времени оседания выводится формулой (`settleTimeAtRestUpperBound` в `spring.ts`), без пробного прогона: `rate = ζ·ω₀` (underdamped) либо `ω₀/(ζ + √(ζ²−1))` (медленный корень overdamped), `t_settle ≤ [ln(1/ε) + max(0, ln ω₀) + ln(max(1, amp))]/rate`. Автономный frame-loop-исполнитель (`validateSpringParams`) требует, чтобы граница помещалась в бюджет кадра-капа (`MAX_FRAMES·FIXED_DT_S`, ≈33.3 с), и бросает `LM091` иначе — ноль коробочных констант, бюджет выведен из порогов контура кадра.

Из того же следует и граница домена (#218): чистому `spring()` frame-loop не нужен, поэтому он обязан вычислять **любую** физически валидную систему — сколь угодно медленную и незатухающую (`ζ = 0`) включительно. Валидирует его только `validateSpringPhysics`: конечные `mass > 0` (`LM088`), `stiffness > 0` (`LM089`), `damping ≥ 0` (`LM090`).

Обратная сторона той же монеты — конструкторы субпутя `./spring`: раз решение замкнуто, замкнуты и **обратные задачи**. `fromVisualDuration` решает точное первое пересечение `x(t) = 1`, `fromPeak` — точное обращение первого перелёта, `fromOscillation` — прямое задание комплексных полюсов. Без итераций и подгонки.

### Catastrophic cancellation и тождество Виета (#226)

Классическая запись медленного overdamped-полюса — `r_slow = −ω₀(ζ − √(ζ²−1))`. При больших `ζ` величины `ζ` и `√(ζ²−1)` совпадают во всех значащих битах, и их разность округляется в ноль: при `ζ ≳ 1e8` полюс **терялся целиком** — кривая замирала, а валидатор ложно отвергал быстро оседающие жёсткие пружины (`rate → 0` ⇒ `t_settle → ∞` ⇒ `LM091`).

Лекарство — тождество Виета для `r² + 2α·r + ω² = 0`: произведение корней `r_slow·r_fast = ω²` точно. Отсюда резольвентная форма без вычитания близких чисел:

```text
split  = √(−Δ)
r_slow = −ω² / (α + split)      // ≡ −α + split, но БЕЗ катастрофического вычитания
r_fast = −α − split
gap    = r_slow − r_fast = 2·split   // точно; вычитание ПОЛЮСОВ у critical
                                     // сокращало бы α-части — cancellation в новом месте
```

Та же скалярная форма — `(ζ − d)(ζ + d) = 1 ⇒ ζ − d = 1/(ζ + d)` — используется в `settleTimeAtRestUpperBound` и в overdamped-ветке `fromVisualDuration`.

Вторая точка потери точности — `t ≈ 0`: разность `e^(r_slow·t) − e^(r_fast·t)` двух почти-единиц округляется в ноль, а при огромных `|v0|` две модальные амплитуды взаимно уничтожаются. Солвер разделяет **конечную базу** и **линейный по v0 вклад** и пишет разности через `expm1`:

```text
V(t)  = −e^(r_slow·t) · expm1(−gap·t) / gap        // = (e^(r_slow·t) − e^(r_fast·t))/gap,
                                                   // предел V → t при t → 0 сохранён
x(t)  = −expm1(r_fast·t) + (r_fast + v0)·V(t)      // −expm1 вместо 1 − exp: малая база не тонет
x′(t) = ω²·V(t) + v0·e^(r_slow·t)·(1 − r_fast·expm1(−gap·t)/gap)
```

В скорости `ω²·V(t)` — снова Виет: точное `r_slow·r_fast = ω²` вместо произведения двух уже округлённых полюсов. Итог: `x′(0) = v0` держится без epsilon-переключателей, а форма одновременно является линейным по `v0` базисом для пакетного сэмплирования каналов.

Поверх сырого солвера `spring()` держит страж финитности `clampFinite`: `NaN → 0` (пружина в старте — безопасный CSS-фоллбек, не `−1.8e308` с неверным знаком), `±Infinity → ±Number.MAX_VALUE`. Инвариант «выход всегда конечен» — контракт, а не пожелание.

### Хвост springAsEasing: конечный горизонт и Hermite-коррекция (#219)

Пружина затухает бесконечно — у easing-функции время конечно: `t ∈ [0, 1]`. `springAsEasing` решает это в два шага.

**Шаг 1 — выведенный горизонт.** В безразмерном времени `u = ω₀·τ` ищется наименьший горизонт `U`, при котором замкнутая **огибающая** решения удовлетворяет

```text
|1 − x(U)| + (4/27)·|U·x′(U)| ≤ tolerance
```

где `tolerance` — settle-допуск пакета (`CONVERGENCE_THRESHOLD`, 0.5% диапазона). Огибающие по ветвям (из кода, в стабильной pole-форме #226; `d = √(ζ²−1)`, `r̂s = 1/(ζ+d)`):

```text
ζ < 1:  e^(−ζu)/√(1−ζ²) · (1 + (4/27)·u)
ζ = 1:  e^(−u) · (1 + u + (4/27)·u²)
ζ > 1:  e^(−r̂s·u) · ((ζ + d + r̂s)/(2d) + (4/27)·u/d)
```

Огибающая экспоненциально затухает и монотонна по хвосту, поэтому `U` находится детерминированной брекет-бисекцией (удвоение верхней границы, затем 64 шага деления) — без wall-clock, без итеративного солвера самой пружины. Критерий безразмерен ⇒ зависит только от `ζ` ⇒ scale-equivalent `(m, k, c)` получают одну кривую. Реальный горизонт: `T = U/ω₀`; сама easing-функция сэмплирует `f(t) = x(t·T)`.

**Шаг 2 — C¹ Hermite-коррекция хвоста.** На горизонте `f(1) = f₁ ≈ 1`, но не ровно `1`, и наклон `s₁ = T·x′(T)` не ровно `0` — старая шкала просто обрезала кривую, и критическая пружина прыгала на ≈5.6% с наклоном ≈0.21 в endpoint. Вместо обрезания хвост запечатывается кубической Hermite-поправкой:

```text
g(t) = f(t) + (1 − f₁)·(3t² − 2t³) − s₁·(t³ − t²)
```

Оба базисных кубика обнуляются вместе с производной в `t = 0`, поэтому старт не тронут; в `t = 1` они дают ровно `g(1) = 1`, `g′(1) = 0`. Итого `g(0) = 0`, `g′(0) = 0`, `g(1) = 1`, `g′(1) = 0` — эндпоинты точны (дисциплина NE2), стыковка с соседними сегментами C¹-гладкая. Поправка ограничена по построению: `max|3t² − 2t³| = 1` и `max|t³ − t²| = 4/27` на `[0, 1]`, откуда

```text
|g − f| ≤ |1 − f₁| + (4/27)·|s₁| ≤ tolerance
```

— именно этим слагаемым `(4/27)·|U·x′(U)|` и обусловлен критерий горизонта из шага 1: горизонт подобран так, чтобы Hermite-хвост заведомо не исказил форму сильнее допуска. В рантайме поправка — два Horner-коэффициента (`g = f + t²·(c2 + t·c3)`), сэмпл — один вызов внутреннего солвера без повторной валидации и аллокаций.

Незатухающая система (`ζ = 0`) конечного горизонта не имеет — огибающая не затухает. Это граница именно этого исполнителя (#218): `springAsEasing` бросает `LM167`, хотя те же параметры валидны для чистого `spring()`.

## API

Все длительности этого субпутя — **секунды** (не миллисекунды); `bounce`, `overshoot` и аргумент `t` easing-функции безразмерны. Все броски — `MotionParamError` с полем `code`; каталог с лечением — [docs/errors.md](../errors.md). Во всех конструкторах `mass` опционален: любое не-число, неконечное или `≤ 0` значение тихо заменяется на `1` (это дефолт, не ошибка), а **результат** проверяется `validateSpringPhysics` — непредставимость точного преобразования конечным положительным double (underflow/overflow `stiffness`/`damping`) даёт честные `LM088`/`LM089`/`LM090`, не тихую подмену (#218).

### fromBounce

```ts
function fromBounce(options: FromBounceOptions): SpringParams;

interface FromBounceOptions {
  readonly duration: number;          // секунды, > 0
  readonly bounce: number;            // ∈ [−1, 1]
  readonly mass?: number | undefined; // по умолчанию 1
}
```

Канон SwiftUI `Spring(duration:bounce:)`. Точное преобразование:

```text
ζ = 1 − bounce,   ω₀ = 2π/duration,   stiffness = m·ω₀²,   damping = 2·m·ζ·ω₀
```

`bounce = 0` — критическое демпфирование, `> 0` — упругая, `< 0` — пере-демпфированная; `bounce = 1` честно означает `ζ = 0` ⇒ `damping = 0`. Возврат: `SpringParams`. Бросает: `LM093` (`duration`), `LM092` (`bounce`); транзитом `LM088`/`LM089`/`LM090`.

### fromVisualDuration

```ts
function fromVisualDuration(options: FromVisualDurationOptions): SpringParams;

interface FromVisualDurationOptions {
  readonly visualDuration: number;    // секунды, > 0
  readonly bounce: number;            // ∈ [−1, 1]
  readonly mass?: number | undefined; // по умолчанию 1
}
```

Время **первого визуального касания цели** ≈ `visualDuration` (класс Motion), `ζ = 1 − bounce`. Для `ζ < 1` — точное решение первого пересечения `x(t) = 1`:

```text
ω₀ = (π − atan(√(1−ζ²)/ζ)) / (√(1−ζ²)·Tv)      // ζ = 0: ω₀ = π/(2·Tv)
```

Для `ζ ≥ 1` пересечения нет — `Tv` трактуется как выход на ~99% цели по медленнейшей моде: `ω₀ = ln(100)·(ζ + √(ζ²−1))/Tv` (стабильная форма корня, #226). Возврат: `SpringParams`. Бросает: `LM093` (`visualDuration`), `LM092` (`bounce`); транзитом `LM088`–`LM090`.

### fromPeak

```ts
function fromPeak(options: FromPeakOptions): SpringParams;

interface FromPeakOptions {
  readonly overshoot: number;         // доля первого перелёта ∈ (0, 1]
  readonly peakTime: number;          // время первого пика, секунды, > 0
  readonly mass?: number | undefined; // по умолчанию 1
}
```

Обратная задача из **наблюдаемых** координат underdamped-системы из покоя (#230) — точное обращение, не пресет:

```text
L = −ln(overshoot),   ζ = L/√(π²+L²),   ω₀ = √(π²+L²)/peakTime
stiffness = m·(π²+L²)/peakTime²  (без sqrt),   damping = 2·m·L/peakTime
```

Один `log`, ноль итераций. `overshoot = 1` честно означает `ζ = 0` (незатухающая, нормализованный пик ровно `2`); `overshoot = 0` не имеет underdamped-прообраза (критический предел) и отклоняется без epsilon-подмены — «без перелёта» описывается `fromBounce({bounce: 0})` или `fromVisualDuration`. Возврат: `SpringParams`. Бросает: `LM171` (`overshoot` вне `(0, 1]`), `LM093` (`peakTime`); транзитом `LM088`–`LM090`.

### fromOscillation

```ts
function fromOscillation(options: FromOscillationOptions): SpringParams;

interface FromOscillationOptions {
  readonly period: number;            // период затухающих колебаний, секунды, > 0
  readonly halfLife: number;          // амплитуда огибающей падает вдвое, секунды, > 0
  readonly mass?: number | undefined; // по умолчанию 1
}
```

Прямое задание комплексных полюсов `p = −α ± i·β` наблюдаемыми координатами (#230):

```text
α = ln2/halfLife,   β = 2π/period,   stiffness = m·(α²+β²),   damping = 2·m·α
```

Всегда underdamped (`β > 0` ⇒ `ζ < 1`), точно; «`period = ∞`» не является скрытой критической ветвью — конечность обязательна. Возврат: `SpringParams`. Бросает: `LM093` (`period`, `halfLife`); транзитом `LM088`–`LM090`.

### springPresets

```ts
const springPresets: Readonly<Record<
  'default' | 'gentle' | 'wobbly' | 'stiff' | 'slow' | 'molasses',
  SpringParams
>>;
```

Канонические пресеты react-spring (tension/friction при `mass = 1`): `default` {1, 170, 26}, `gentle` {1, 120, 14}, `wobbly` {1, 180, 12}, `stiff` {1, 210, 20}, `slow` {1, 280, 60}, `molasses` {1, 280, 120}. Объект и каждый пресет заморожены (`Object.freeze`) — пин контракта. Не бросает.

### springAsEasing

```ts
function springAsEasing(params: SpringParams): (t: number) => number;
```

Пружина как easing-функция нормализованного времени `t ∈ [0, 1] → value` (форма OVERSHOOTING при `ζ < 1`). Математика — в разделе [#219 выше](#хвост-springaseasing-конечный-горизонт-и-hermite-коррекция-219): шкала `t = 1` — выведенный конечный горизонт, хвост запечатан C¹ Hermite-коррекцией, `|g − f|` ограничен settle-допуском. Возвращённая функция: `t ≤ 0 → 0`, `t ≥ 1 → 1`, `NaN → 0` (NE1/NE2), неконечный промежуточный результат → `1`; не бросает. Валидация и горизонт — один раз в конструкторе. Бросает (конструктор): `LM088`/`LM089`/`LM090` (физика), `LM167` (`ζ ≤ 0`: незатухающая пружина не имеет конечного easing-горизонта; лечение — `damping > 0` либо живой исполнитель `drive`/`MotionValue`).

### Type-only экспорты

`FromBounceOptions`, `FromVisualDurationOptions`, `FromPeakOptions`, `FromOscillationOptions`. Типы `SpringParams`/`SpringResult` — из корня `@labpics/motion`.

## Контракты

Только то, что заявлено в коде/JSDoc:

- **SSR-safe / zero-DOM.** Солвер и весь субпуть `./spring` — чистая математика: без DOM, `window`, часов и глобального состояния; безопасно на сервере и в воркерах.
- **Детерминизм.** Идентичный вход → идентичный выход; easing-горизонт — детерминированная бисекция по замкнутой огибающей, wall-clock не используется.
- **Финитность.** `spring()` всегда возвращает конечные `value`/`velocity` (страж `clampFinite`: `NaN → 0`, `±Infinity → ±MAX_VALUE`); функция из `springAsEasing` всегда возвращает конечное число; конструкторы либо возвращают конечные положительные `SpringParams`, либо бросают `MotionParamError` рано.
- **Точность (#218/#226/#230).** Конструкторы — точные математические преобразования запрошенных координат без коэрсии под чей-либо бюджет; численная форма солвера выбрана против catastrophic cancellation; представимость проверяется честной ошибкой, не подменой.
- **Единицы.** Все длительности — секунды; `t` солвера — секунды; `velocity` — нормализованные единицы позиции в секунду.
- **Reduced-motion.** Этот слой `prefers-reduced-motion` не читает — уважение reduced-motion принадлежит исполнителям (`drive` и выше).

## Примеры

Полюса напрямую из наблюдаемых координат + произвольный доступ по `t`:

```ts
import { spring, type SpringParams } from '@labpics/motion';
import { fromOscillation } from '@labpics/motion/spring';

// p = −α ± i·β: α = ln2/halfLife (огибающая), β = 2π/period (частота).
const params: SpringParams = fromOscillation({ period: 0.8, halfLife: 0.25 });

// Замкнутая форма — каждая точка независима, интегратор и «предыдущий кадр» не нужны.
for (const t of [0, 0.2, 0.4, 0.8, 1.6]) {
  const { value, velocity } = spring(params, t); // t — секунды
  console.log(t, value.toFixed(4), velocity.toFixed(4));
}
```

Обратная задача в замкнутой форме и C¹-easing с Hermite-хвостом:

```ts
import { fromPeak, springAsEasing } from '@labpics/motion/spring';

// Наблюдаемые координаты: первый перелёт 12% ровно в t = 0.3 s. Один log, ноль итераций.
const params = fromPeak({ overshoot: 0.12, peakTime: 0.3 });

// ease(0) = 0, ease(1) = 1, ease′(1) = 0 — хвост запечатан Hermite-коррекцией (#219).
const ease = springAsEasing(params);
const el = document.querySelector('.card') as HTMLElement;
const durationMs = 700; // шкала нормализованного времени — забота вызывающего
const start = performance.now();

function frame(now: number): void {
  const t = Math.min((now - start) / durationMs, 1);
  el.style.transform = `scale(${1 + 0.2 * ease(t)})`;
  if (t < 1) requestAnimationFrame(frame);
}
requestAnimationFrame(frame);
```

Границы точности и домена: экстремальный overdamped (#226) и незатухающий `ζ = 0`:

```ts
import { MotionParamError, spring, type SpringParams } from '@labpics/motion';
import { springAsEasing } from '@labpics/motion/spring';

// ζ = c/(2√(km)) = 5·10⁸ — прежняя форма −ω₀(ζ−√(ζ²−1)) теряла медленный полюс,
// и кривая замирала (#226). Резольвентная форма Виета считает его точно.
const extreme: SpringParams = { mass: 1, stiffness: 1, damping: 1e9 };
console.log(spring(extreme, 1e9).value); // медленно, но честно движется к 1

// ζ = 0 валиден для чистого spring(), но не имеет конечного easing-горизонта.
try {
  springAsEasing({ mass: 1, stiffness: 100, damping: 0 });
} catch (e) {
  if (e instanceof MotionParamError && e.code === 'LM167') {
    // лечение: damping > 0 либо живой исполнитель (drive/MotionValue)
  }
}
```
