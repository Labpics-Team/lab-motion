# Процесс релиза

> Роль: практическое руководство по выпуску публичной версии `@labpics/motion`.

## Источники правды

| Факт | Источник |
|---|---|
| Версия пакета | `package.json#version` |
| Состав артефакта | `package.json#files` и `package.json#exports` |
| Изменения версии | `CHANGELOG.md` |
| Неизменяемая точка релиза | git-тег `vX.Y.Z` |
| Опубликованный артефакт | npm registry |

Версия, тег и заголовок changelog обязаны совпадать.

## SemVer до `1.0.0`

| Изменение | Версия |
|---|---|
| Исправление без изменения публичного контракта | patch |
| Совместимая новая возможность | minor |
| Несовместимое изменение публичного API, поведения или токенов | minor с разделом миграции |

После `1.0.0` несовместимое изменение требует major.

Публичным контрактом считаются:

- runtime-экспорты и типы;
- значения и имена токенов;
- ошибки и правила валидации;
- lifecycle `play/pause/seek/cancel/stop/destroy`;
- SSR, reduced-motion и fallback-поведение;
- заявленная поддержка runtime и браузеров.

## Подготовка release PR

1. Закрыть целевой release issue.
2. Обновить `package.json#version`.
3. Перенести пользовательские изменения из `Unreleased` в `CHANGELOG.md` под `## [X.Y.Z] - YYYY-MM-DD`.
4. Добавить миграцию для несовместимых изменений.
5. Выполнить обязательные гейты:

```bash
pnpm install --frozen-lockfile
pnpm typecheck
pnpm build
pnpm test
pnpm size
pnpm pack:smoke
node scripts/check-release.mjs vX.Y.Z
```

6. Проверить содержимое тарбола и импорт через ESM/CJS.
7. Получить независимое review release PR.

## Публикация

1. Смержить release PR в `main`.
2. Создать GitHub Release из неизменяемого тега `vX.Y.Z`.
3. Workflow `Release` повторно проверяет версию, changelog и все publish-гейты.
4. Публикация выполняется с npm provenance.
5. Проверить установку версии в чистом проекте.

Ручная публикация с рабочей машины не является штатным путём.

## Отмена и исправление

Опубликованная версия не переписывается.

- Ошибка без несовместимости исправляется новой patch-версией.
- Ошибочный breaking contract исправляется новой minor-версией до `1.0.0`.
- Скомпрометированный релиз помечается deprecated в npm; удаление применяется только при подтверждённой необходимости.
- Тег опубликованной версии не перемещается и не переиспользуется.
