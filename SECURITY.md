# Политика безопасности

## Поддерживаемые версии

Исправления безопасности выпускаются для последней опубликованной версии пакета. При необходимости исправление переносится в поддерживаемую предыдущую линию отдельным решением сопровождающих.

## Сообщение об уязвимости

Не создавайте публичный issue с эксплуатационными деталями.

Используйте **GitHub Security Advisories → Report a vulnerability** в этом репозитории. Сообщение должно содержать:

- затронутую версию;
- минимальный сценарий воспроизведения;
- ожидаемое и фактическое поведение;
- оценку влияния;
- известные способы эксплуатации;
- предлагаемое исправление, если оно есть.

## Обработка

Сопровождающие:

1. подтверждают получение сообщения;
2. воспроизводят дефект в закрытом контуре;
3. определяют затронутые версии и границы влияния;
4. добавляют регрессионный тест;
5. выпускают исправление и advisory;
6. раскрывают детали после доступности безопасной версии.

Сроки не обещаются до воспроизведения и оценки риска. Статус передаётся автору сообщения через приватный advisory.
