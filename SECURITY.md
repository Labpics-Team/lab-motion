# Безопасность

## Поддерживаемые версии

Исправления безопасности выпускаются для последней опубликованной версии и актуальной ветки `main`. Для более старых версий сначала проверяется воспроизводимость на поддерживаемой версии.

## Сообщение об уязвимости

Не публикуйте сведения об уязвимости в открытом issue.

Используйте приватный отчёт GitHub:

1. откройте вкладку **Security** репозитория;
2. выберите **Report a vulnerability**;
3. приложите минимальное воспроизведение и описание влияния.

Отчёт должен содержать:

- затронутую версию или commit;
- требуемые условия эксплуатации;
- пошаговое воспроизведение;
- наблюдаемый результат;
- возможное влияние на потребителя;
- известный обходной путь;
- предложение исправления, если оно есть.

## Область безопасности

К уязвимостям относятся, в частности:

- выполнение непредвиденного кода через входные данные;
- загрязнение прототипа;
- небезопасная запись CSS или DOM-атрибутов;
- утечка ресурсов, приводящая к устойчивому отказу в обслуживании;
- обход заявленного SSR- или sandbox-контракта;
- компрометация цепочки публикации или npm-артефакта.

Обычные ошибки анимации, несовпадение визуального результата и предложения API оформляются как публичные Issues.

## Координированное раскрытие

Детали остаются приватными до выпуска исправления или согласованного решения о публикации. Исправление включает регрессионный тест, оценку затронутых версий и release note без эксплуатационных деталей, которые преждевременно повышают риск.
